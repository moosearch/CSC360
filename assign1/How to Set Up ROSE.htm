Installing Cygwin for ROSE

Cygwin

Cygwin is a UNIX-like execution environment written for Windows platform. It provides all the familiar UNIX utilities (e.g., ls, more, cat, ..., and gcc). Note that all binaries generated by Cygwin GCC compiler must be run inside Cygwin because it provides a Win32 compatible library linked with each executable (.exe) file. (Note: If you choose to do your ROSE development on Linux, then the entire GCC toolchain is already available, you don't need to install Cygwin. However, you may still need to install JDK, followed the instructions below. At this moment, we still have some minor problem of running the ClassLinker of SimpleRTJ.)

Download Cygwin (called "setup.exe") here:

  • http://www.cygwin.com/

Run setup.exe. Click Next. Install from Internet. If you are on the lab machines, set the install directory to your H drive and install for "Just me" (i.e. H:\cygwin).

For download site,  you could choose

  • http://cygwin.mirrors.hoobly.com

but it should not really matter which one you choose.

We will need some packages. Use the search bar at the top to speed things up, they should all be under the Devel heading. Where it says skip, click it once and it should change to a version number with a checked box under "Bin?". Now, here they are:

  • libncurses-devel: (devel) libraries for terminal handling
  • gcc-core: C compiler
  • make: (The GNU version of the 'make' utility)
  • ctags: (A C programming language...)

You may also install one of your favoriate text editor (e.g., vim). If you missed any package that you would like to install, just run "setup.exe" again using the same initial directory that you install in the beginning. It remembers all the settings and packages that you have installed locally.

If you are on the lab machines, copy the Cygwin Terminal shortcut on the desktop and put in your H Drive. You will probably want to run Cygwin from this shortcut, and who knows if it will be on the desktop the next time you login.


ROSE and Java

Grab a copy of "assign1.org.tgz" (which includes ROSE) and put it in your home folder (i.e. C:\cygwin\home\<username> or H:\cygwin\home\<username>).

Start up cygwin (hopefully using the desktop shortcut). Then type:

tar xzvf assign1.org.tgz   % this will unarchive everything
cd assign1
cd code
cd rose
make clean
make

If everything works, then your GCC compiler is installed correctly, otherwise check your bash shell $PATH variable.

If there are complaints about permission denied: the folder could be read only or cygwin might not have administrative privileges. In order to run ROSE, you also need a copy of the Java compiler from Oracle (formerly SUN).

You need to install the Java SDK (or JDK) from http://www.oracle.com/technetwork/java/javase/downloads/jdk7-downloads-1880260.html. You may choose 32-bit (x86) or 64-bit (x64) version. We are not running the standard JVM. We only use the "javac" compiler to generate the class files (the byte code).

In order to run "javac",  first check where JDK is installed. Very likely it will be at:

cd "/cygdrive/c/Program Files/Java/jdk1.7.0_21/bin"

Then, edit ".bashrc" (or ".bash_profile"). At the bottom of the file append:

export PATH="/cygdrive/C/Program Files/Java/jdk1.7.0_21/bin":$PATH
where "jdk1.7.0_21" is the version of your java installed location. Please note the order of the new $PATH. Restart "cygwin" shell. Check whether "javac" is in your PATH:
which javac
/cygdrive/c/Program Files/Java/jdk1.7.0_21/bin/javac
If you get an error message, then you have installed Java somewhere else. Just update your Java bin $PATH accordingly.

Running ROSE on Linux (Not Supported)

Although ROSE had been tested on Linux, we don't have the human resources to maintain and test both Windows and Linux versions. If you must use Linux or OS X, then we recommend that you install VirtualBox or any other VM emulator on your machine and then run Windows XP or 7 inside a virtual machine. (Note: We haven't tested this particular installation. We don't guarantee that it will work, i.e., whether the Pentium Performance Counter is supported in a virtual machine environment.)
